# 📄 PRD - 外贸独立站（B2B）

> **文档版本**: v1.0
> **创建日期**: 2026-02-24
> **产品类型**: B2B 外贸询盘型独立站
> **Demo 产品**: 木纤维砧板（Wood Fiber Cutting Board）

---

## 一、项目背景与目标

### 1.1 项目背景

- 业务模式：**B2B** 外贸，面向海外采购商 / 批发商
- 创始人背景：产品设计师，拥有完整的 2D 工程图纸（DWG/PDF）及 3D 模型（Rhino / NX）
- 核心差异化：通过网站在线展示 **2D 图纸** 和 **3D 交互模型**，体现专业设计与制造能力

### 1.2 项目目标

| 目标 | 说明 |
|------|------|
| **专业展示** | 以木纤维砧板为 Demo，在线展示产品的 2D 工程图纸和 3D 可旋转模型 |
| **高效获客** | 通过 SEO 获取自然流量，引导客户提交询盘（RFQ） |
| **自动入档** | 客户询盘自动归档到 Google Sheets，邮件实时通知 |
| **多语言** | 中英双语，覆盖目标市场 |
| **低成本运营** | 零月费部署（Vercel），几乎零维护负担 |

---

## 二、目标用户

| 用户角色 | 描述 |
|---------|------|
| **海外采购商 / 买手** | 在 Google 搜索供应商，浏览产品后提交询盘 |
| **品牌商 / 经销商** | 寻找 OEM/ODM 合作伙伴，关注设计与定制能力 |
| **网站管理员（你自己）** | 管理产品信息、查看和跟进客户询盘 |

## 三、技术架构

### 3.1 技术选型

| 层级 | 技术 | 选型理由 |
|------|------|---------|
| **框架** | Next.js 14 (App Router) | SSG/SSR 对 SEO 极友好，内建 API 路由 |
| **语言** | TypeScript | 类型安全，AI 辅助编码效率最高 |
| **样式** | Tailwind CSS | 开发速度快，AI 最熟练的 CSS 方案 |
| **3D 展示** | react-three-fiber + @react-three/drei | Three.js 的 React 封装，开箱即用 |
| **2D 展示** | react-zoom-pan-pinch | 图纸缩放/平移/全屏查看 |
| **多语言** | next-intl | Next.js 官方推荐 i18n 方案 |
| **表单** | react-hook-form + zod | 表单处理与验证 |
| **邮件** | nodemailer | 询盘邮件通知发送到 Gmail |
| **数据归档** | Google Sheets API | 客户询盘自动写入 Sheets |
| **部署** | Vercel | 免费额度充足，全球 CDN，自动 HTTPS |

### 3.2 架构模式

```
┌─────────────────────────────────────────────────┐
│                   Vercel (CDN)                   │
│  ┌───────────────────────────────────────────┐   │
│  │           Next.js 静态页面 (SSG)            │   │
│  │  ┌─────┐ ┌──────┐ ┌──────┐ ┌───────┐     │   │
│  │  │ 首页 │ │产品列表│ │产品详情│ │关于我们│     │   │
│  │  └─────┘ └──────┘ └──────┘ └───────┘     │   │
│  │           ┌──────────────┐                 │   │
│  │           │  3D/2D 查看器  │                 │   │
│  │           └──────────────┘                 │   │
│  └───────────────────────────────────────────┘   │
│  ┌───────────────────────────────────────────┐   │
│  │         API Routes (Serverless)            │   │
│  │  ┌──────────┐                              │   │
│  │  │ /api/inquiry │ ──→ Gmail + Google Sheets │   │
│  │  └──────────┘                              │   │
│  └───────────────────────────────────────────┘   │
└─────────────────────────────────────────────────┘
```

### 3.3 安全策略

| 措施 | 说明 |
|------|------|
| **静态部署** | 无后台、无数据库，攻击面极小 |
| **速率限制** | 询盘 API 限频，防恶意刷请求 |
| **输入验证** | zod 严格校验所有表单输入 |
| **reCAPTCHA** | Google reCAPTCHA v3 防机器人 |
| **CORS** | 仅允许自身域名调用 API |
| **HTTPS** | Vercel 自动配置 SSL 证书 |

## 四、页面结构与功能需求

### 4.1 站点地图

```
/                        ← 首页（Landing Page）
├── /products            ← 产品列表页
│   └── /products/[id]   ← 产品详情页（含 3D/2D 查看器）
├── /about               ← 关于我们
├── /contact             ← 联系我们 / 询盘页
└── /api/inquiry         ← 询盘提交 API（后端）
```

> 所有页面均支持 `/en/...` 和 `/zh/...` 多语言路由。

---

### 4.2 首页 (Landing Page)

**目标**：第一印象建立信任，引导用户浏览产品或直接询盘。

| 模块 | 内容 | 交互 |
|------|------|------|
| **Hero Banner** | 全屏大图/视频 + 一句话品牌卖点 + CTA 按钮 | "View Products" / "Get a Quote" |
| **产品亮点** | 3-4 个卖点图标 + 描述（天然材质 / 抗菌 / 可定制 / 环保） | 悬浮动效 |
| **精选产品** | 3-6 个产品卡片（图片 + 名称 + 简介） | 点击跳转产品详情 |
| **设计能力展示** | 设计流程图 / 工具展示（Rhino/NX） | — |
| **信任背书** | 认证标志 / 合作客户 Logo / 工厂照片 | — |
| **CTA 区域** | "Ready to start? Contact us today" | 跳转询盘页 |

---

### 4.3 产品列表页

**目标**：清晰展示所有产品，支持分类筛选。

| 模块 | 内容 |
|------|------|
| **分类筛选** | 按类型（方形/圆形）、用途、材质等筛选 |
| **产品网格** | 卡片式布局：产品图 + 名称 + 简要规格 + "View Details" 按钮 |
| **排序** | 按名称 / 上新时间排序 |

---

### 4.4 产品详情页 ⭐（核心页面）

**目标**：全方位展示产品，促使客户提交询盘。

| 模块 | 内容 | 说明 |
|------|------|------|
| **产品图片画廊** | 多角度产品实拍图，支持放大查看 | Lightbox 灯箱效果 |
| **🔴 3D 模型查看器** | 嵌入式 3D 交互窗口 | 客户可 360° 旋转、缩放、拖拽查看模型 |
| **🔴 2D 图纸查看器** | 工程图纸展示 | 支持缩放、平移、全屏查看 |
| **产品描述** | 材质、工艺、特点等文字介绍 | — |
| **规格参数表** | 尺寸、重量、材质、颜色、包装等 | 表格形式 |
| **MOQ & 交期** | 最低起订量、生产周期、包装方式 | — |
| **询盘 CTA** | "Request a Quote" 按钮 | 打开询盘弹窗，自动带入当前产品信息 |
| **相关产品** | 推荐其他同类产品 | 卡片横向滚动 |

#### 3D 查看器功能要求

- 支持 **.glb 格式**模型加载（Rhino / NX 导出）
- 鼠标/触屏操作：**旋转、缩放、平移**
- 预设环境光照、地面阴影
- 加载动画（模型文件较大时）
- 响应式适配（移动端可用手势操作）

#### 2D 图纸查看器功能要求

- 支持 **高清 PNG / SVG** 格式
- 鼠标滚轮/双指缩放
- 拖拽平移
- 全屏查看按钮
- 重置视图按钮

---

**目标**：建立品牌信任感，展示专业能力。

| 模块 | 内容 |
|------|------|

### 4.6 联系我们 / 询盘页

**目标**：低门槛引导客户提交询盘，减少流失。

#### 询盘表单字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 公司名称 | Text | ✅ | — |
| 联系人姓名 | Text | ✅ | — |
| 邮箱 | Email | ✅ | 需验证格式 |
| 电话 / WhatsApp | Text | ❌ | 含国际区号 |

#### 提交后流程

```
客户提交询盘
    ↓

    ↓
┌───────┬──────────┬──────────────┐
│       │          │              │
↓       ↓          ↓              ↓
📧     📊         📬             📱
Gmail   Google     自动回复客户     (后期)
通知    Sheets     确认邮件        WhatsApp
管理员   自动归档                   通知
```
